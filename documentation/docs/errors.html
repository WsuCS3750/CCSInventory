Error Logging

Errors are caught by surrounding 

try{} catch{}
blocks, and logged. The user is then directed to errorpages/error.aspx

Special consideration must be give when redirecting users via 

Response.Redirect()
because that method works by generating a redirection exception which is inadvertantly caught by error logging.

The error log can be viewd at log/view-errorlog.aspx

Sample Usage

try
{
    // DO WORK
    Response.Redirect('new-page.aspx');
}
catch (System.Threading.ThreadAbortException) { }
catch (Exception ex)
{
    LogError.logError(ex);
    Response.Redirect("../errorpages/error.aspx");
}
